1. Split Indications and Backfill

将完整 Excel 分子、药品或资产池中的多适应症字段，机械拆分为“一行一个适应症”，并通过公式回填全部原始字段，为后续疾病 Mapping 建立完整、可追溯的逐适应症底稿。

核心价值

原始数据库经常在一个单元格中记录多个适应症。若直接进行疾病 Mapping，容易出现一个来源行对应多个疾病、判断粒度不一致、回填错位等问题。

本 Skill 解决的是数据结构问题：

把多适应症记录拆成逐项记录；

保留空适应症，不因缺失信息丢失来源行；

保留重复适应症，不擅自去重；

保留完整原始数据和稳定主键；

使用公式从完整来源表回填字段；

为下游 Mapping 提供统一的“一行一个适应症”输入。

它不进行任何疾病标准化、ICD 判断、TA 分类或医学 Mapping。

适用场景

适用于：

一条药品、分子或资产记录包含多个获批适应症；

适应症由分号或其他已确认分隔符连接；

后续需要逐适应症开展 Disease Mapping；

需要保留原始字段、公式、数据类型和审计链。

不适用于：

已经是一行一个适应症的数据；

需要从长句中识别疾病实体；

需要清洗、翻译或标准化适应症；

需要判断适应症属于哪个 TA；

需要合并同一 Disease 或跨数据库资产。

核心处理规则

对每个非空适应症片段生成一行，保留原始顺序。

去除片段前后的空格，但不重写其医学表达。

不对重复片段去重；重复内容保留并在检查报告中提示。

原适应症为空时，仍保留一条拆分结果为空的记录。

连续分隔符或尾部分隔符产生的空片段不生成额外行，但必须报告。

稳定主键按原始 Excel 类型读取和写入，不把数值型 ID 擅自改为文本。

完整原始适应症继续保留在来源 Sheet 和更新版拆分底稿中。

更新版拆分底稿除前三个结构字段外，其余字段通过 XLOOKUP 从完整来源 Sheet 回填。

公式应保留真实空值，不能把空白回填为 0。

标准流程

1. 只读检查

检查：

文件哈希、大小和更新时间；

Sheet 列表、隐藏 Sheet、名称定义；

有效行列范围；

必需字段是否存在；

空白或重复表头；

空白或重复主键；

空白实体名称；

空适应症；

实际包含分隔符及真正产生多行的记录；

连续分隔符、尾部分隔符和重复适应症；

合并单元格；

跨 Sheet 或外部公式依赖。

此阶段不生成工作簿。

2. 样例预览

在正式生成前展示 1–2 个代表性例子，优先包括：

一条确实会拆成多行的记录；

一条空适应症记录；

一条实际 XLOOKUP 公式示例。

必须由使用者确认输入 Sheet 是完整来源池，并确认拆分效果。

3. 正式生成

在明确批准后生成新工作簿，并绑定已批准输入文件的 SHA-256 和来源签名。若输入文件在批准后发生变化，必须停止，不能静默继续。

4. 重新打开验证

保存后重新打开输出文件，检查 Sheet、行数、列数、公式覆盖、来源保持和回填结果。

输出

标准输出工作簿严格包含三个 Sheet，并按以下顺序排列：

1. 完整来源 Sheet

保留原 Sheet 名称；

保留原始值、公式和数字格式；

作为唯一事实来源。

2. 拆分结果

包含：

稳定主键；

实体名称；

适应症拆分结果。

用于快速核阅拆分结果。

3. 更新版拆分底稿

包含：

稳定主键；

实体名称；

适应症拆分结果；

完整来源表中的其他字段。

除结构字段外，其余字段通过公式从完整来源 Sheet 回填。

质量门槛与停止条件

以下情况必须停止：

输入文件或来源 Sheet 不明确；

输入不是完整来源池；

必需字段缺失或重复；

表头为空或重复；

稳定主键为空或重复；

实体名称为空；

有效区域存在合并单元格；

非空适应症只包含分隔符或空格；

分隔符含义不明确；

来源公式依赖将被标准三 Sheet 输出删除的其他 Sheet；

输入和输出扩展名不一致；

输出将覆盖输入文件或已有文件；

批准后文件哈希或来源签名变化；

公式覆盖、来源保持或重新打开验证失败。

本 Skill 是 Mapping 前的数据结构准备步骤：

下游：map-indications-to-ta-diseases；

不负责：疾病 Mapping、同 Disease 合并、上市库与临床库合并。

跨 TA 复用原则

可跨 TA 复用的是：

一行一个适应症；

稳定主键与完整来源保持；

空适应症不删除；

拆分前预览和人工确认；

公式回填和重新打开 QC。

不可跨 TA 固化的是：

输入行数和拆分后行数；

字段数量；

来源 Sheet 名称；

具体分隔符；

具体药品、适应症或业务结果。
